# Culture Amp Connector — Setup Guide

## Overview

The Culture Amp connector syncs engagement surveys, employee records, and performance goals from your Culture Amp account into Shielva. It authenticates via a Bearer token against the Culture Amp REST API.

---

## Step 1 — Generate a Culture Amp API Token

1. Log in to Culture Amp as an account administrator.
2. Navigate to **Settings → Integrations → API Access**.
3. Click **Generate New Token** (or **Add Token**).
4. Give the token a descriptive name (e.g. "Shielva Connector").
5. Copy the token — it is shown only once.
6. Click **Save**.

> **Note:** The token inherits the permissions of the account used to generate it. Use an admin account to access all surveys, employees, and goals.

---

## Step 2 — Gather your install field

| Field | Where to find it | Example |
|-------|-----------------|---------|
| **API Token** | Generated in Step 1 | `ca_live_abc123...` |

---

## Step 3 — Install in Shielva ACP

1. In the Shielva ACP, navigate to **Integrations → Add Connector → Culture Amp**.
2. Paste your **API Token** into the install field.
3. Click **Install**. Shielva calls `GET /v1/surveys` to verify the token.
4. On success, the connector status shows **ONLINE**.

---

## Required Permissions

The API token must have access to:

- **Surveys** — read access to engagement surveys
- **Employees** — read access to employee directory
- **Goals** — read access to performance goals
- **Reviews** — read access to performance reviews
- **Groups** — read access to departments and teams

Administrators have full access by default.

---

## API Endpoints Used

| Resource | Method | Endpoint |
|----------|--------|----------|
| Me / Health | GET | `/v1/me` |
| Surveys | GET | `/v1/surveys` |
| Single Survey | GET | `/v1/surveys/{id}` |
| Employees | GET | `/v1/employees` |
| Goals | GET | `/v1/goals` |
| Performance Reviews | GET | `/v1/performance/reviews` |
| Groups | GET | `/v1/groups` |

All requests use: `Authorization: Bearer {api_token}`

Base URL: `https://api.cultureamp.com`

---

## Authentication Details

Culture Amp uses Bearer token authentication:

```
Authorization: Bearer <your_api_token>
```

The connector injects this header automatically on every request. You never need to construct auth headers manually.

## Troubleshooting

### 401 Unauthorized

**Cause:** The API token is invalid or has been revoked.

**Fix:**
- Confirm the token was copied correctly (no extra spaces or newline characters).
- Regenerate the token at **Settings → Integrations → API Access**.

---

### 403 Forbidden

**Cause:** The token does not have permission to access the requested resource.

**Fix:**
- Ensure the token was generated by an admin account.
- Check that the account has access to the specific resource (surveys, employees, goals).

---

### 404 Not Found

**Cause:** The requested resource ID does not exist in Culture Amp.

**Fix:**
- Verify the survey ID or other resource identifier in the Culture Amp UI.

---

### 429 Too Many Requests (Rate Limit)

**Cause:** Culture Amp enforces rate limits on API requests.

**Fix:**
- The connector automatically retries with exponential backoff (up to 3 attempts).
- If 429s persist, reduce sync frequency.

---

### Sync returns 0 surveys or employees

**Cause:** The token account has no data in scope, or the organization has no active surveys/employees.

**Fix:**
- Confirm active surveys exist in the Culture Amp dashboard.
- Use an admin token with access to all organizational data.
