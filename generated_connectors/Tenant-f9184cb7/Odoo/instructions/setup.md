# Setup Instructions: Odoo

## Overview

The Odoo connector integrates your organization's Odoo ERP/CRM instance with the Shielva platform over Odoo's standard JSON-RPC API. Once connected, Shielva can read partners, leads, sale orders, customer invoices and the product catalog — and can create or update records on your behalf. Authentication uses an Odoo **API key** generated by a regular Odoo user, so you never share your account password with Shielva.

The connector works for both Odoo.com SaaS instances (`https://mycompany.odoo.com`) and self-hosted deployments. It speaks to a single Odoo database — install one connector per database if you have multiple.

---

## Prerequisites

Before you begin, make sure you have:

- An **Odoo instance** at a URL Shielva can reach (Odoo.com SaaS or a self-hosted server)
- The exact **database name** as shown on the Odoo login page (above the Email / Password fields)
- An **Odoo user account** with permission to access the models you want to surface (Contacts, CRM, Sales, Invoicing, Inventory, etc.)
- The ability to generate an **API Key** from that user's preferences (Odoo 14+ supports per-user API keys natively)

---

## Step-by-Step Configuration

### Step 1: Odoo Instance URL (`instance_url`) — **Required**

1. Sign in to your Odoo instance in a browser.
2. Copy the URL from the address bar, dropping any path (e.g. `https://mycompany.odoo.com`).
3. Paste this value into the **Odoo Instance URL** field in Shielva.

> The connector appends `/jsonrpc` automatically. Do not include it yourself.

---

### Step 2: Database name (`db`) — **Required**

1. Log out of Odoo, or open the login page in a private window.
2. Above the **Email** / **Password** fields the login page shows the database name in a small dropdown labelled "Database". Copy that exact value.
3. Paste it into the **Database name** field in Shielva.

> For Odoo.com SaaS instances the database name is usually the subdomain of your URL (e.g. `mycompany` for `https://mycompany.odoo.com`). Self-hosted deployments may use any identifier — copy whatever Odoo displays.

---

### Step 3: Username (`username`) — **Required**

1. Use the email address of the Odoo user whose API key you will generate in Step 4.
2. The user must have permission on every Odoo model you intend to query — at minimum **Contacts / Read**.

---

### Step 4: API Key (`api_key`) — **Required**

1. Sign in to Odoo as the user from Step 3.
2. Click the user avatar in the top-right corner and choose **Preferences** (or **My Profile** in some Odoo versions).
3. Open the **Account Security** tab.
4. Click **New API Key**. Enter a label (e.g. `Shielva`) and click **Generate Key**.
5. Odoo shows the API key **only once**. Copy it now.
6. Paste it into the **API Key** field in Shielva. The field is stored encrypted.

> **Common mistake:** Do not paste your Odoo password into this field. When two-factor authentication is enabled on the user, Odoo rejects JSON-RPC logins that present a password — only API keys work. Using an API key also keeps your password out of Shielva even when 2FA is off.

---

### Step 5: Rate Limit (`rate_limit_per_min`) — **Optional**

- **Default value:** `60` (requests per minute)
- Odoo has no hard public rate-limit; the default keeps traffic gentle on shared instances. Raise this for high-volume sync operations on dedicated servers.

---

## Completing the Install

After filling in the four required fields, click **Connect** in the Shielva connector dashboard. Shielva calls `common.authenticate` on your Odoo instance to verify the credentials. A successful call returns a numeric `uid`, which the connector caches for subsequent requests.

---

## Testing the Connection

1. After install completes, the connector status badge should show **Connected** (green).
2. Click **Run Health Check** — Shielva re-authenticates and reports the current `uid`.
3. Click **APIs → list_partners** and run it with `limit=10` to confirm read access to `res.partner`.

---

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `Odoo rejected the supplied credentials` | Wrong db / username / api key | Re-check Step 2 (exact db name) and re-generate the API key (Step 4). |
| `Odoo authentication failed: AccessDenied` | API key revoked or user disabled | Generate a new API key in Odoo → Preferences → Account Security. |
| `Odoo access denied: AccessError` on a list call | The user lacks ir.model.access rights on the target model | Grant the user the relevant Odoo group (e.g. **Sales / User** for `sale.order`). |
| `Could not reach Odoo instance: ...` | Wrong instance URL or network egress blocked | Open `<instance_url>/web/login` in a browser from the Shielva network. |
| Health check works but `list_invoices` returns `[]` | Filter excludes inbound / vendor bills | The connector filters on `move_type='out_invoice'` (customer invoices). Use `execute_method` with a custom domain for other move types. |
| `Odoo RPC error (...)` on a write call | Missing required Odoo field (e.g. company_id) | Use `execute_method` with the exact `values` dict Odoo expects, or extend the connector method to fill in the default. |
